LinkNest — AI가 링크를 대신 읽어주는 개인 지식 라이브러리

공유 버튼 한 번으로 AI가 요약·분류·저장. LinkNest가 만들어가는 개인 링크 큐레이션 서비스를 소개합니다.
March 28, 2026 · 2 min · LinkNest

개요#

인터넷을 쓰다 보면 나중에 읽으려고 저장해둔 링크들이 쌓입니다. 하지만 현실은 다시 열어보는 경우가 드뭅니다. 저장했다는 사실만 기억날 뿐, 어디에 있는지도 모르게 됩니다.

LinkNest는 이 문제를 해결하기 위해 만들고 있는 개인 링크 큐레이션 서비스입니다.

해결하는 문제#

링크 저장 방식의 근본적인 문제는 저장과 소화가 분리되어 있다는 점입니다.

  • 저장은 쉬운데 나중에 다시 열기가 귀찮다
  • 쌓인 링크가 많아질수록 관리가 불가능해진다
  • 원본을 다시 방문해야만 내용을 확인할 수 있다
  • 분류와 태그를 직접 해야 해서 저장 자체가 번거로워진다

결국 대부분의 사람들이 링크를 저장해두고 읽지 않는 루프에 빠집니다.

핵심 흐름#

LinkNest는 공유 버튼 한 번으로 모든 것을 처리합니다.

어떤 앱이든 → 공유 버튼 → LinkNest 선택
→ AI 자동 처리: 본문 파싱 → 요약 생성 → 카테고리 분류 → 태그 생성
→ 원본 재방문 없이 정리된 내용 바로 확인

저장하는 순간 AI가 읽기 가능한 형태로 가공합니다. 사용자는 아무것도 하지 않아도 됩니다.

핵심 가치#

가치설명
저장 마찰 최소화공유 버튼 한 번으로 완료
저장 즉시 소화저장되는 순간 AI가 가공
자동 지식 라이브러리분류·태그·요약이 자동 완성

기술 스택#

LinkNest는 다음 기술로 만들어지고 있습니다.

  • Flutter — iOS/Android 크로스플랫폼 앱
  • Supabase — 인증, 데이터베이스, 실시간 처리
  • AI API — 링크 콘텐츠 요약 및 분류

MVP는 모바일 앱에 집중합니다. Share Intent로 어떤 앱에서든 링크를 받아 처리하는 것이 핵심입니다.

플랫폼 로드맵#

단계플랫폼
MVPiOS / Android 앱
MVP서비스 소개 웹사이트
Post-MVP웹앱 (전체 기능)
Post-MVP브라우저 익스텐션

이 블로그에서 다룰 것들#

개발하면서 배운 것들을 솔직하게 공유할 예정입니다.

  • Flutter — Share Intent 구현, 상태관리, UI 설계
  • Supabase — 인증 연동, Row Level Security, 실시간 처리
  • AI 연동 — 링크 파싱, 요약 프롬프트 설계, 비용 최적화
  • 인프라 — CI/CD, GitHub Actions, 배포 자동화

완성된 결과물보다 과정과 결정의 이유를 중심으로 씁니다.

Key Takeaways#

  • LinkNest는 “저장하지만 읽지 않는” 링크 문제를 AI로 해결하는 서비스
  • 공유 버튼 → AI 자동 처리 → 즉시 소화 가능한 형태로 저장
  • Flutter + Supabase 기반 MVP를 만들어가는 과정을 이 블로그에 기록
