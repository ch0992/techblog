## Overview

대부분의 사람들은 안티그래비티(AntiGravity)를 단순한 '코딩 챗봇'으로 사용합니다. 하지만 진정한 파워 유저는 이를 **'소프트웨어를 찍어내는 공장'**으로 활용합니다. 

구글의 새로운 AI 코딩 환경은 클로드 4.5 소넷, 제미나이 3 프로, GPT-4.5 등 현존하는 최강의 모델들을 하나의 오케스트라처럼 지휘할 수 있는 능력을 갖추고 있습니다. 본 포스팅에서는 잭 로버츠(Jack Roberts)가 제안한 **FLOW 프레임워크**를 통해, 아이디어 구상부터 실제 배포까지 이어지는 완벽한 엔지니어링 파이프라인을 구축하는 법을 배웁니다.

---

## Background: 왜 99%의 사람들은 AI 코딩에 실패하는가?

AI에게 단순히 "대시보드 하나 만들어줘"라고 요청하면, 결과물은 조잡하거나 핵심 기능이 빠진 '조(Joe) 앱'이 되기 십상입니다. 성공적인 프로덕트 빌딩을 위해서는 **질문(Chatting)**이 아니라 **설계(Engineering)**가 필요합니다. 

안티그래비티는 파일 탐색기, 코드 에디터, 에이전트 매니저가 유기적으로 연결된 환경입니다. 이 환경을 100% 활용하기 위해서는 정교한 단계별 접근(Framework)이 필수적입니다.

---

## Solution: 전문가를 위한 4단계 FLOW 프레임워크

### 1단계: F (Frame) - 문제의 정의와 구체화
아인슈타인은 "문제를 해결하는 데 1시간이 주어진다면 55분은 문제를 정의하는 데 쓰겠다"고 했습니다. 
- **Claude와의 대화**: 안티그래비티 내부에서 클로드(Claude) 모델을 선택하여 아이디어를 정교하게 다듬습니다. 내 생각을 도전(Challenge)하게 만들고, 발생할 수 있는 엣지 케이스를 미리 파악합니다.
- **SOP 생성**: 정제된 아이디어를 바탕으로 **500자 이내의 작업 표준 지침(SOP)**을 작성합니다. 이 SOP는 이후 에이전트가 흔들리지 않고 작업하게 만드는 나침반이 됩니다.

### 2단계: L (Layout) - 시각적 영감과 브랜드 가이드
코딩 전, 앱의 'Look & Feel'을 결정하는 단계입니다.
- **이미지 업로드**: Dribble 등에서 찾은 디자인 영감을 직접 안티그래비티에 붙여넣습니다. 제미나이 3 프로는 이미지를 코드로 변환하는 데 압도적인 성능을 보입니다.
- **gemini.md 활용**: 프로젝트 루트에 `gemini.md` 또는 브랜드 가이드 파일을 생성합니다. 폰트, 컬러 팔레트, 타이포그래피 규칙을 명시해두면 에이전트가 이를 참조하여 디자인 정합성을 유지합니다.

### 3단계: O (Orchestration) - 에이전트 지휘와 구축
안티그래비티의 핵심인 **오케스트레이션** 단계입니다.
- **멀티 에이전트 가동**: 에이전트 매니저를 열고 여러 에이전트를 동시에 실행합니다. (예: 에이전트 A는 법적 규제 조사, 에이전트 B는 경쟁사 분석, 에이전트 C는 코어 로직 개발)
- **주석(Annotation) 피드백**: 코드나 아티팩트의 특정 라인에 직접 주석을 달아 피드백을 줍니다. "이 부분은 제공한 브랜드 가이드를 더 강조해줘"라고 명확히 지시하세요.
- **자기 치유(Self-Healing)**: 안티그래비티의 에이전트는 직접 브라우저를 열어 UI를 클릭하고 테스트하며 오류를 스스로 수정합니다.

### 4단계: W (World) - 세상에 내놓기 (Deploy)
완성된 코드는 로컬 컴퓨터(`localhost`)에만 머물러서는 안 됩니다.
- **GitHub 연결**: MCP 서버를 통해 GitHub를 연동합니다. 안티그래비티가 직접 리포지토리를 만들고 코드를 푸시하도록 명령합니다.
- **Vercel 호스팅**: GitHub 리포지토리를 Vercel과 연결하여 단 몇 초 만에 실제 도메인으로 앱을 배포합니다.

## Deep Dive: 작업 성격에 최적화된 모델 선택 전략

안티그래비티의 강점은 상황에 따라 엔진(Model)을 갈아 끼울 수 있다는 것입니다.

| 작업 유형 | 추천 모델 | 이유 |
| :--- | :--- | :--- |
| **디자인 & UI** | **Gemini 3 Pro** | 구글 소프트웨어 최적화 및 뛰어난 시각 분석 능력 |
| **자연어 & 카피라이팅** | **Claude 4.5 Sonnet** | 사람에 가장 가까운 자연스러운 문체와 사고력 |
| **코드 검수 & 리팩토링** | **Claude 4.5 Sonnet** | 논리적 정밀도와 토큰 효율성 |
| **단순 반복 업무** | **GPT-4.5** | 안정적이고 경제적인 처리 속도 |

## 심화: MCP 서버와 커스터마이징 (Gachas)

### **⚠️ 효율적인 도구 관리 (Tool Management)**
안티그래비티에서 MCP 서버를 통해 도구를 추가할 때, **한 번에 50개 이상의 도구가 활성되지 않도록** 주의하세요. 컨텍스트 윈도우 효율을 위해 정말 필요한 도구(약 20개 내외)만 활성화하는 것이 품질 향상의 비결입니다.

### **비장의 카드: `Context 7` MCP**
최신 문서와 라이브러리 정보를 실시간으로 가져오기 위해 `Context 7` MCP 설치를 강력히 권장합니다. 이는 AI가 '옛날 방식'으로 코딩하는 문제를 원천 봉쇄합니다.

## 마치며

안티그래비티는 단순한 도구가 아니라 여러분의 **'AI 직원'**들이 일하는 사무실입니다. **FLOW 프레임워크**를 통해 각 에이전트에게 명확한 역할을 부여하고, 브랜드 가이드라는 규격을 제시하며, 마지막 배포까지 자동화하세요. 이 시스템이 안착되는 순간, 여러분의 개발 속도는 이전과는 비교할 수 없는 수준에 도달할 것입니다.
